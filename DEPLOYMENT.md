# Deployment Guide

## Prerequisites

1. Node.js 14+ installed
2. npm or yarn installed
3. GitHub account with repository access

## Setup Steps

### 1. Install Dependencies

```bash
cd redlining-dashboard
npm install
```

### 2. Prepare Data Files

Place the following JSON files in `public/data/`:
- `frost-bank-data.json`
- `webster-bank-data.json`
- `all-bank-data.json`
- `summary-stats.json`

These files should be generated by running `process_existing_data.py` (once the Excel file is available).

### 3. Test Locally

```bash
npm start
```

The app will open at `http://localhost:3000`

### 4. Build for Production

```bash
npm run build
```

This creates an optimized production build in the `build/` directory.

### 5. Deploy to GitHub Pages

#### Option A: Using gh-pages package (Recommended)

1. Install gh-pages:
```bash
npm install --save-dev gh-pages
```

2. Add to package.json scripts:
```json
"predeploy": "npm run build",
"deploy": "gh-pages -d build"
```

3. Deploy:
```bash
npm run deploy
```

#### Option B: Manual Deployment

1. Build the app: `npm run build`
2. Copy contents of `build/` to `gh-pages` branch
3. Push to GitHub

### 6. Configure GitHub Pages

1. Go to repository Settings â†’ Pages
2. Select source: `gh-pages` branch
3. Select folder: `/ (root)`
4. Save

The dashboard will be available at:
`https://[username].github.io/redlining-dashboard`

## Troubleshooting

### Data Not Loading
- Ensure JSON files are in `public/data/`
- Check browser console for errors
- Verify file paths in App.tsx match actual file names

### Routing Issues
- Set `homepage` in package.json to match GitHub Pages URL
- Use `HashRouter` instead of `BrowserRouter` if needed

### Build Errors
- Check Node version: `node --version` (should be 14+)
- Clear node_modules and reinstall: `rm -rf node_modules && npm install`
